Description

COBILL — The Solution

A web-based invoicing app for French freelancers, built with the two most reliable technologies for the job:

🏦

COBOL (1959)

The language of banks. Handles all financial calculations with native decimal arithmetic. Zero rounding errors.

🖨

PostScript (1985)

The language of printers. Generates pixel-perfect PDF invoices. No library, no dependency.

⚡

HTMX (2020)

14 KB of JavaScript. Dynamic UI without React, Vue, or any framework. Zero build step.

"Your bank runs on COBOL. Your printer runs on PostScript. Your invoicing app should too."

Architecture

How It Works

BROWSER AWS EC2 (Ubuntu) +-----------+ +----------------------------+ | HTML+HTMX | HTTP POST/GET | Apache 2.4 + mod_cgi | | + CSS | ----------------> | | | (14 KB) | <-- HTML fragment | +--------------------+ | +-----------+ | | COBOL PROGRAMS | | | | | | | | invoice.cob | | | | > PIC 9(5)V99 | | | | > HT / TVA / TTC | | | | | | | | pdf-gen.cob | | | | > Write .ps file | | | | > Call Ghostscript | | | | > Return PDF link | | | +--------------------+ | | | | | +----------v---------+ | | | ISAM | PDF/ | | | | .dat | .ps .pdf | | | +---------+----------+ | +----------------------------+

Data Flow

  • User fills invoice form
  • HTMX sends POST to CGI
  • Apache invokes COBOL binary
  • COBOL computes exact totals
  • COBOL writes PostScript file
  • Ghostscript converts to PDF
  • HTML fragment returned to browser
  • HTMX injects result — no reload

No JSON. No API layer. No virtual DOM.
The server renders HTML. Period.

Challenges Identified

Risks & Mitigations

Challenge Risk Mitigation
COBOL + CGI integration Medium ACCEPT FROM ENVIRONMENT + DISPLAY — prototype on Day 1-2
Form data parsing in COBOL Medium Reusable cgi-utils.cob module (~50 lines), written once
PostScript learning curve Low Only ~15 commands needed. Invoice template is ~150 lines
COBOL calling Ghostscript Low CALL "SYSTEM" — one line of COBOL
Session management Medium Cookie + ISAM session store. Deferred to Week 3
Dev environment (macOS → Linux) Low Develop directly on EC2 via SSH + VS Code Remote

Fallback: If CGI proves problematic, a thin Python wrapper invokes COBOL binaries. The COBOL calculation engine stays intact regardless.
